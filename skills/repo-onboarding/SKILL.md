# Repo Onboarding

Use this skill when you need a fast first-pass understanding of an unfamiliar repository.

## Goals

- identify the main entrypoints
- find the setup commands
- map the key docs and configuration files
- summarize likely work areas before making changes

## Workflow

1. Read the top-level `README.md`.
2. List the top-level directories and identify the main app or package folders.
3. Find common setup files such as `package.json`, `pyproject.toml`, `Cargo.toml`, `go.mod`, `docker-compose.yml`, or `.env.example`.
4. Identify test and verification commands.
5. Summarize the repo in a short, implementation-oriented note.

## Output

Provide:

- a one-paragraph repo summary
- the likely entrypoints
- the main setup and test commands
- any obvious gaps or missing docs
